## 二、新增功能清单

### 🛡️ 安全加固（6 项）

#### S1 — XSS + CSRF 全面防护

**当前问题**：XSS 过滤器仅覆盖 `/system/*,/monitor/*,/tool/*`；CSRF 被 `csrf.enabled: false` 禁用。

**实现方案**：
1. 修改 `FilterConfig.java`，XSS 过滤器的 `urlPatterns` 加上 `/admin/*,/student/*,/teacher/*`
2. `application.yml` 中 `csrf.enabled` 改为 `true`
3. 所有 AJAX POST 请求统一在 `beforeSend` 中携带 `X-CSRF-TOKEN`
4. 前端模板中统一对用户输入调用 `$.common.escapeHtml()`

#### S2 — BCrypt 密码升级

**当前问题**：学生/教师密码使用 MD5 无盐单次迭代。

**实现方案**：
1. 引入 `BCryptPasswordEncoder`
2. 新增/修改的密码直接使用 BCrypt 加密
3. 登录验证时先 BCrypt 匹配，失败回退旧 MD5 匹配，成功后自动升级为 BCrypt（透明迁移）
4. `password_hash` 列从 varchar(128) 扩展到 varchar(255)

#### S3 — 登录失败锁定

**当前问题**：学生/教师登录无暴力破解防护。

**实现方案**：
1. 使用缓存记录每个账号登录失败次数
2. 连续 5 次失败后锁定 30 分钟
3. 成功后重置计数

#### S4 — 验证码正式启用

**当前问题**：`ShiroConfig.java` 硬编码 `setCaptchaEnabled(false)`。

**实现方案**：
1. 移除硬编码，恢复从 `application.yml` 读取配置
2. 确认登录页数学验证码正常渲染

#### S5 — 安全审计 AOP

**当前问题**：敏感操作无自动记录。

**实现方案**：
1. 新建 `@SecurityLog` 注解 + 切面，自动记录 IP、操作类型、目标对象
2. 覆盖操作：登录、改成绩、删数据、改密码、审核操作
3. 管理员安全审计页面查看和搜索日志

#### S6 — 安全仪表盘

**当前问题**：管理员无安全态势概览。

**实现方案**：
1. 管理员首页增加安全概览卡片：今日登录失败次数、异常登录数、最近安全事件、锁定账户列表

### 📋 业务功能（10 项）

#### B1 — 教师开课申请 + 审核

**链路**：教师 → 管理员

**实现方案**：
1. 课程表新增 `status`（0=待审核, 1=已通过, 2=已驳回）和 `reject_reason` 字段
2. 教师端"开课申请"页面 → 管理员"课程审核"标签页 → 通过/驳回（填写理由）
3. 审核结果通过通知中心（B6）通知教师
4. 管理员直接创建的课程默认状态为"已通过"

#### B2 — 选课时间窗口

**链路**：管理员 → 学生

**实现方案**：
1. 利用 `sys_config` 表存储选课起止时间
2. 管理员配置时间窗口，学生选课/退课时校验，窗口外禁止
3. 前端窗口外选课按钮禁用并提示

#### B3 — 课程公告

**链路**：教师 → 学生

**实现方案**：
1. 新建 `course_notice` 表（notice_id, cno, tno, title, content, create_time）
2. 教师端：发布/编辑/删除公告；学生端：查看已选课程公告；首页展示最近公告

#### B4 — 学生评教

**链路**：学生 → 教师

**实现方案**：
1. 新建 `course_evaluation` 表（eval_id, sno, cno, tno, semester, rating(1-5), comment, is_anonymous）
2. 学生端：对已选课程评分+评价，支持匿名；教师端：查看汇总（平均分、评价列表）
3. 同一学生同一课程只能评一次

#### B5 — 成绩申诉

**链路**：学生 → 教师 → 管理员

**实现方案**：
1. 新建 `grade_appeal` 表（appeal_id, sno, cno, semester, reason, status, reply, handle_time）
2. 学生端：发起申诉（理由）；教师端：处理（同意/驳回+回复）
3. 管理员端：查看全部申诉记录（监督）
4. 结果通过通知中心通知学生

#### B6 — 消息通知中心

**链路**：全员（互动枢纽）

**实现方案**：
1. 新建 `notification` 表（notif_id, recipient_type, recipient_id, title, content, is_read, create_time）
2. 各事件触发时创建通知记录
3. 顶部导航栏未读消息数角标（所有角色）；消息列表页：按时间倒序、已读/未读标记
4. 支持"全部标记已读"

#### B7 — 图表可视化仪表盘

**链路**：管理员

**实现方案**：
1. 引入 ECharts.js CDN
2. 统计分析页面新增图表视图：选课人数柱状图、成绩分布饼图、院系对比、GPA 分布
3. 数据接口从现有 StatisticsService 扩展

#### B8 — 成绩单 PDF 导出

**链路**：学生

**实现方案**：
1. 使用 Apache PDFBox 生成 PDF
2. PDF 内容：学校信息、学生信息、各学期成绩明细、GPA、打印日期
3. 学生成绩页面新增"下载成绩单 PDF"按钮

#### B9 — 学分统计

**链路**：学生

**实现方案**：
1. 学生端新增"学分统计"板块：总学分要求、已修学分、在修学分、必修/选修达标情况
2. 配合进度条可视化展示

#### B10 — 个人信息 + 密码修改

**链路**：学生/教师

**实现方案**：
1. 新增"个人信息"编辑页面（非敏感字段）+ "修改密码"页面
2. 密码复杂度校验：最小 8 位，含大小写+数字
