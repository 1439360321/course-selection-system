# 选课系统 验收演示

## 环境

| 项目 | 值 |
|------|-----|
| 地址 | `http://localhost:8080` |
| 数据库 | MySQL 8.x — `course_selection` |

|------|--------|------|

| 学生 | `2021001001` | `123456` |

## 10 项要求实现方式

| # | 要求 | 实现方式 |
|---|------|---------|
| 1 | B/S 结构 | 前端 Thymeleaf 模板 + Bootstrap/jQuery 在浏览器运行，后端 Spring Boot 在服务器运行，通过 HTTP 协议通信 |
| 2 | School 数据库修改 | 在 MySQL 中新建 `course_selection` 库，自行设计 8 张业务表（department/admin/student/teacher/course/class/courseselection/audit_log），外键约束 + 3NF 规范化 |
| 3 | 不同角色不同权限 | Shiro 框架实现 RBAC：`sys_role` 定义 admin/teacher/student 三个角色，Controller 层 `@RequiresRoles` 注解限制访问，菜单表 `sys_menu` 按角色分配不同菜单 |
| 4 | 管理员学分制管理 | 7 个 Controller：DepartmentController、StudentController、TeacherController、CourseController（含学分/学时字段）、ClassController（排课）、CourseSelectionController（选课记录）、StatisticsController（统计分析） |
| 5 | 学生选课查询 | StudentBusinessController：浏览课程 → `course/select` 选课（含重复校验+容量限制）→ `course/my` 已选列表 → `course/drop` 退课 → `grade/list` 成绩 → `gpa` 绩点 |
| 6 | 教师成绩录入 | TeacherBusinessController：`course/list` 授课课程 → `course/students/list` 选课学生 → `grade/save` 录入平时+考试成绩（总评=平时×40%+考试×60% 实时计算） |
| 7 | 统计分析 | 视图驱动：`v_course_enrollment`（选课统计）、`v_fail_students`（不及格）、`v_student_grades`（成绩单）；存储过程 `sp_student_gpa`（GPA）；三个角色各有统计入口 |
| 8 | 触发器+存储过程 | 3 个触发器：`trg_course_delete`/`trg_student_delete`/`trg_teacher_delete` 级联删除；1 个存储过程 `sp_student_gpa`，Java 端通过 MyBatis `CALLABLE` 调用（CourseSelectionMapper.xml → `callSpStudentGpa`） |
| 9 | MySQL 数据库 | MySQL 8.0.39，InnoDB 引擎，utf8mb4 编码，Druid 连接池 |
| 10 | 辅助功能 | `audit_log` 审计日志记录管理员操作；Druid `/druid` SQL 监控；数学验证码防爆破；SpringDoc `/swagger-ui.html` API 文档；若依自带用户/角色/菜单管理 |


## 演示流程（15 min）

### 1. 数据库设计（2 min）
Navicat 展示 8 张表：department → admin → student → teacher → course → class → courseselection → audit_log
> 亮点：总评=平时×40%+考试×60%，实时计算不存储，满足 3NF

### 2. 管理员
登录 `admin / admin123`
- 学院/学生/教师 CRUD，课程管理（学分、学时、必修/选修）
- 授课排课（教师+课程+学期+时间）
- 统计分析（选课统计、不及格统计、GPA 查询）

### 3. 学生端
登录 `2021001001 / 123456`
- 浏览课程 → 选课 → 防重复 → 退课
- 我的选课 → 成绩查询 → GPA

### 4. 教师端
登录 `T2021001 / 123456`
- 我的课程 → 选课学生列表
- 录入平时成绩 + 考试成绩 → 总评自动计算

### 5. 数据库高级特性
- 触发器演示：删除课程 → 选课和授课记录自动清空
- 存储过程：`CALL sp_student_gpa('2021001001', @gpa);`
- 视图：v_course_enrollment、v_fail_students 等 4 个

### 6. 辅助功能
- `/druid` SQL 监控（ruoyi / Zjr20050124） http://localhost:8080/druid
- `/swagger-ui.html` API 文档
- audit_log 审计日志
