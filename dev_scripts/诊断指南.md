# 选课系统无法显示数据 - 诊断指南

## 📋 问题描述
- 页面无法显示数据
- 搜索功能无法使用

---

## 🔍 排查步骤

### 1. 检查浏览器控制台错误
按 F12 打开开发者工具，查看 Console 标签页：

**可能出现的错误：**
- `404 Not Found` - 接口路径错误
- `403 Forbidden` - 权限不足
- `500 Internal Server Error` - 服务器内部错误
- `Network Error` - 网络连接问题

### 2. 检查网络请求
在开发者工具的 Network 标签页中：

**查找 `/admin/course/list` 请求：**
- 检查状态码（应该是 200）
- 检查 Response 内容
- 检查 Request Headers（是否包含 Cookie）

### 3. 检查后端日志
查看项目启动日志，查找错误信息：

**常见错误：**
```
Unknown column 'xxx' in 'field list'
Table 'xxx' doesn't exist
Access denied for user
```

## 🐛 可能的原因和解决方案

### 原因1：前端JavaScript错误
**症状：** 页面加载但数据不显示

**解决方案：**
1. 打开浏览器控制台查看错误
2. 检查 `course.html` 中的 JavaScript 代码
3. 确认 jQuery 和 Bootstrap 是否正确加载

### 原因2：后端接口返回错误
**症状：** Network 标签显示 500 错误

**解决方案：**
1. 检查后端日志
2. 检查 Mapper XML 中的 SQL 语句
3. 验证数据库表结构是否匹配

### 原因3：权限问题
**症状：** Network 标签显示 403 错误

**解决方案：**
1. 确认登录成功
2. 检查 `sys_role_menu` 表中的权限配置
3. 确认 admin 用户拥有课程管理权限

### 原因4：数据库连接问题
**症状：** 无法连接数据库或查询超时

**解决方案：**
1. 检查 `application-druid.yml` 中的数据库配置
2. 确认 MySQL 服务正在运行
3. 测试数据库连接：
```bash
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -h localhost -u root -p123456 -e "SELECT 1"
```
